data-view
声明特定的DOM元素为视图。一个视图必须有一个DOM元素与之对应，且DOM元素的id即为视图的编号。尝试通过API：View.ofId(viewId)获取一个没有DOM元素与之对应的视图时，View.js将会报错。
true
任一DOM节点
下一节
data-view-default
标明特定视图为默认视图。默认视图是指页面打开后将要呈现的第一个视图。如果没有指定默认视图，则最终呈现DOM顺序上的第一个视图。
true
任一视图节点（声明了data-view属性，且取值为true的DOM节点）
下一节
attr_data-view-group
用于指明视图所隶属的视图群组。
任意字符串
任一视图节点
下一节
data-view-directly-accessible
配置视图是否可以直接访问。一个视图是否能够直接访问（直接访问，是指是否允许用户以手动更改地址栏中地址的方式主动访问特定视图）
true：可以通过直接访问；false：不能直接访问。
HTML节点，或任一视图节点
下一节
data-view-fallback
用于指定视图不能直接访问时，将要呈现的回退视图编号。可以指定一个同样不能直接访问的视图，此时将递归搜索，直到找到一个可以直接访问的回退视图。如果搜索链上的所有视图都不能直接访问，则最终呈现默认视图。
任一存在的视图编号，亦即对应的DOM元素的ID；或伪视图：:default-view；或视图群组：~groupName
任一视图节点
下一节
data-view-rel-disabled
禁用触摸跳转。设置为true后，声明了data-view-rel属性的节点将无法响应用户的触摸操作。
true：禁止跳转；false：允许跳转
任一声明了data-view-rel属性的DOM节点
下一节
data-view-rel
响应用户触摸，跳转到指定的目标位置。支持伪视图和视图群组。支持视图选项，如：<span data-view-rel=”category!id=1”>分类</span>
任一存在的视图编号，亦即对应的DOM元素的ID；或伪视图：":back"（后退），":forward"（前进），":default-view"（默认视图）；或视图群组；或其它链接。当配置为视图群组时，需要添加特殊符号：“~”，如：<span data-view-rel="~category">分类</span>。当配置为其它链接地址时，需要添加特殊符号：“@”，如：<span data-view-rel="@index.html">首页</span>，<span data-view-rel="@http://www.google.com.tw">谷歌</span>。
任一DOM节点
下一节
data-view-rel-type
声明通过data-view-rel属性切换视图时使用的视图切换类型。
nav：以"压入历史堆栈"的方式切换；change：以"替换当前堆栈"的方式切换
任一声明了data-view-rel属性的DOM节点
下一节
data-view-title
用于声明视图标题。声明了视图标题的视图在进入时，View.js将使用视图标题更新文档标题。
任意字符串

android：安卓操作系统；ios：苹果操作系统
