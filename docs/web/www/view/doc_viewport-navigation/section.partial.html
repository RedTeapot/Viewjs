View.js支持如下几种方式进行视图跳转： 1) A链接，如：
商品详情
此时，要求视图：goods-detail为默认视图或可以直接访问。 2) data-view-rel属性，如：
商品详情 商品详情
个人中心
其中”@”符号用于告诉View.js进行页面跳转访问。 此时，要求视图：goods-detail为默认视图或可以直接访问。 3) JS API：View.navTo，如：
View.navTo(“setting”, {params: {a: 1}, options: {b: 2}})
4) JS API：View.changeTo，如：
View.changeTo(“setting”, {params: {a: 1}, options: {b: 2}})
其中，1)和2)两种方式无法传参；3)和4)两种方式可以传参。 View.navTo和View.changeTo的差别之处，在于是否向浏览堆栈中添加记录。使用View.navTo执行视图A向视图B的跳转时，用户可以在视图B中通过物理返回键返回A；使用View.changeTo执行视图A向视图B的跳转时，A的浏览历史将被B所替代，用户在视图B中无法通过物理按键返回A。
