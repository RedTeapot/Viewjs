data属性配置参考手册
data属性是HTML5新增的，供应用程序定义自己的属性前缀，如：data-abc，data-nav等。 View.js提供了如下几个配置：
  • data-view
  • data-view-default
  • data-view-rel
  • data-view-rel-disabled
  • data-view-directly-accessible
  • data-view-fallback
data-view
用于指明对应的DOM节点是一个视图
true, false
无
<div data-view = "true">...</div>
HTMLElement("div")、HTMLElement("section")等
data-view-default
用于指明对应的视图是一个默认视图
true, false
同一html源文件中，最多只能有一个视图指定此属性并赋值为true
<div data-view = "true" data-view-default = "true">...</div>
HTMLElement("div")、HTMLElement("section")等
data-view-rel
用于定义元素点击（触摸）后要定向到的视图
  • [目标视图ID（不包含字符：“#”）]：定向到的目标视图的id
  • :back：定向到浏览器回退一次后对应的视图
  • :forward：定向到浏览器前进一次后对应的视图
无
<a data-view-rel = "my-bill">我的账单</a>
HTMLElement("div")、HTMLElement("section")等
data-view-rel-disabled
禁用元素点击（触摸）时的视图定向功能
true, false
无
<a data-view-rel = "my-bill" data-view-rel-disabled = "true">我的账单</a>
HTMLElement("div")、HTMLElement("section")等
data-view-directly-accessible
定义对应的视图是否可以通过地址栏hash直接访问
true, false
此属性可同时配置于html节点和视图节点上。如果同时配置，则视图节点的优先级更高
<html data-view-directly-accessible = "false"> <div data-view = "true" data-view-directly-accessible = "true">...</div> /html>
HTMLElement("div")、HTMLElement("section")等
data-view-fallback
定义视图不能直接访问时需要呈现的视图ID
[目标视图ID（不包含字符：“#”）]：视图不能直接访问时需要呈现的视图ID
对于不可以直接访问的特定视图，如果没有配置回退视图则呈现默认视图； 如果配置了回退视图且回退视图可以直接访问，则呈现回退视图； 如果配置的回退视图也不支持直接访问，则呈现默认视图。亦即，回退视图也支持定义回退视图
<html data-view-directly-accessible = "false"> <div data-view = "true" data-view-directly-accessible = "true">...</div> /html>
HTMLElement("div")、HTMLElement("section")等
title
定义视图特有的标题。可以忽略。如果视图设置有此属性，则视图进入后插件会自动将文档标题设置为视图标题，视图离开后自动恢复文档标题。注：初始的文档标题为文档加载完成后的标题
[标题字符串]
<html data-view-directly-accessible = "false"> <div data-view = "true" data-view-title = "标题1">...</div> /html>
HTMLElement("div")、HTMLElement("section")等
